Introduction

The term Geospatial refers to either an implicit or explicit reference to a location relative to the earth.

Metadata is information which captures the basic characteristics of a resource. It represents the who, what, when, where, why, and how of the resource.

The term Geospatial Intelligence or GEOINT encompasses the exploitation and analysis of imagery and geospatial information to describe, assess, and visually depict physical features and geographically referenced activities on the Earth. GEOINT consists of imagery, imagery intelligence, and geospatial information.

A GEOINT Standard is a documented agreement containing technical specifications or other precise criteria to be used consistently as rules, guidelines, or definitions of characteristics to ensure that materials, products, processes, or services are fit for the analysis and visual representation of physical features and geographically referenced activities. NGAs responsibility is to functionally manage standards for GEOINT to achieve interoperability, shared awareness, and the ensuing decision superiority envisioned through net-centric operations.

The meaning and contexts of the preceding terms build upon one another in the order shown. Familiarity with these terms and their contexts are critical and provide the foundation for understanding and applying the NMF and NMIS companion specification in the many communities that comprise the NSG.

The NMF defines a profile of ISO 19115:2003/Cor 1:2006 for specifying geospatial metadata in the NSG. The NSG includes the integration of technology, policies, capabilities, and doctrine necessary to conduct geospatial intelligence in a multi-intelligence environment; it includes the DoD and non-DoD components of the IC, including, where appropriate, coalition and Federal and civil agency partners.

This profile of the NMF defines the minimum set of NMF elements required to publish and discover reports (documents) developed by GEOINT analysts.

NSG Metadata Foundation GEOINT Analytic Products Profile

1       Scope

1.1    General

Geospatial metadata is used in the National System for Geospatial Intelligence (NSG) to specify the structure and to describe the characteristics of, Geospatial Intelligence (GEOINT) data. Geospatial metadata describes the geospatial data elements used to specify information about features, events, coverages (including images and elevation surfaces), datasets (collections of such information), measurements, observations, and services (capabilities to create, access, display, and/or manipulate such information).

One form of GEOINT data is analytic products. These are softcopy documents (Word, PowerPoint, PDF, etc.) which convey the results of an analytic effort. Consisting primarily of free text, these products are difficult to catalog and discover. At the same time, the demands of the analytic workflow make manual entry of detailed metadata tags impractical. A balance is required between the need for rich metadata to support discovery and minimum impact on the production flow. The elements defined in this specification represent that balance. They define the minium elements necessary for the discovery of analytic products. At the same time, most of these elements can be produced through automated means, minimizing the data entry requirements for the analyst.

It must be emphasized that this Profile defines the minimum metadata set such as would be produced by the most impoverished production environment. Rich metadata improves the discovery of analytic products, and thererfore the value of the analytic effort. There are many additional NMF elements which can be populated given the software tools to do so. Developers of analytic production environments should seek to maximize automated population of additional NMF elements through the use of automated data analysis techniques.

1.2    Relation to other Standards

This specification is a profile of the NSG Metadata Foundtion standard. It defines no extensions to the base standard. Some elements which are optional in the NMF may be mandatory in this Profile.

This specification does not have a corresponding Implementation Specification. Implementors should use the NMIS as the implementation specification until such time as an implementation specification for this Profile is developed.

5       Conformance and Compliance

This specification is a profile of the NMF standard. To be compliant with this Profile, an implementation must first meet the compliance criteria for the NMF. In addition, the implementation must demonstrate that the following elements which are optional in the NMF are mandatory in their implementation:

1)     CI_ResponsibleParty shall include the following elements which are optional in the NMF:

a.     organisationName -> NGA

b.    individualName -> employee ID or division and branch

c.     phone OR electronicMailAddress -> one or the other but not both

2)     A Publication Date-Time shall be included in the resource citation

3)     A Unique Identifier is mandatory

4)     A Theme Keyword is mandatory

5)     A Place Keyword is mandatory

6)     A Bounding Box is mandatory

7)     A Geographic Description for a country is mandatory

6       Requirements

Lisa Spuria, the director of the NGA Analysis & Production Division, issued her P Gram on September 10, 2012. This memorandum required all NGA analysts to tag their analytic products with a minimum set of NMF metadata elements as implemented in the CASi ON-Line Publishing Form. Those elements are the foundational requirements for this Profile.

However, the P Gram left some ambiguity as to how the elements presented through the form align with NMF elements. A mapping of each element to their corresponding NSG Metadata Implemetation Specification (NMIS) XPath is provided to remove any uncertainty.

7       GEOINT Analytic Products Profile

This profile defines the minimum subset of NMF elements which are required for analytic products. Those elements are defined in Table 1 and Table 2. Table 1 describes the elements which are defined in the NMF standard. Table 2 describes the elements which are defined by the Intelligence Community security marking standards and included in the NMF by reference. These tables are organized as follows:

Roles

There are three roles in the analytic workflow where the metadata for analytic products may be populated. The elements in Table 1 are organized around those roles. The roles are:

         Creator: the analyst who generates the analytic product.

         Publisher: the person or system which creates the metadata record and makes the product available for discovery.

         Metadata Custodian: the person or organization which is responsible for maintaining the metadata.

In addition, there are eight elements which are required to support discovery and access in the enterprise. These elements reflect characteristics of the production system and are not likely to change from product to product. These elements should be populated by the publishing system and should not require any human intervention.

ID

An identifier which allows an element to be identified elsewhere in the document. Sub-elements are indicated using alphabetic subscripts.

Name

Specifies the name of the element. The names used in this Profile are aligned with the P Gram and may not be the same as those used in the NMF. A mapping of these names to the NMF is provided through the name to XPath mapping provided in Appendix A.

Obligation

Obligation specifies whether a metadata element is Mandatory, Optional, or Conditional. The following rules shall be used to interprete this field.

1)     Mandatory elements defined in the root element shall be populated.

2)     If an optional or conditional element is populated, all mandatory elements defined within the scope of that element shall be populated.

3)     If an optional or conditional element is not populated, none of the elements within the scope of that element shall be populated.

Maximum Occurrence

Specifies the maximum number of instances the metadata entity or the metadata element may have. Single occurrences are shown by 1; repeating occurrences are represented by N. Fixed number occurrences other than one are allowed, and will be represented by the corresponding number (i.e. 2, 3etc).

Multiplicity should always be understood to apply to the specified metadata element as a whole regardless of the nature (e.g., complex) of its value domain. Conversely, multiplicity should always be understood to be only applicable (meaningful) within the scope of the entity within which the element is specified.

Description

A description of the metadata entity/element. Any rules for applying conditional elements are described here as well.
